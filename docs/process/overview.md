# Process

The **`Process`** class is the core abstraction in **jawm**.  
A workflow in jawm is a Python script that defines one or more `Process` objects and executes them. Each `Process` represents a **single computational task**, such as running a script, or launching a containerized/slurm job.

Processes can be chained together through dependencies to form a complete workflow.

---

# What is a Process?

A `Process` represents a **unit of work** primarily with:

- a **name**
- a **script or command to execute**
- optional **dependencies** and **conditional execution**
- configurable **execution environment**
- configurable **execution backend** such as local, Slurm, or Kubernetes
- optional **variables and parameters**

A minimal workflow may consist of a single process, while complex workflows can contain many processes with dependencies.

Example:

```python
import jawm

p = jawm.Process(
    name="hello_world",
    script="""#!/bin/bash
echo "Hello from jawm"
"""
)

p.execute()
```

When executed, jawm will:

1. Resolve configuration parameters
2. Prepare the execution environment
3. Execute the script using the selected backend
4. Store logs and metadata  
      
With the default log level `INFO`, this `hello_world` Process would generate logs similar to followings:  
```text

[2026-03-13 09:52:44] - INFO - hello_world|cb6bc9hopa :: Log folder for process hello_world: /jawm/example/logs/hello_world_20260313_095244_cb6bc9hopa
[2026-03-13 09:52:44] - INFO - hello_world|cb6bc9hopa :: Preparing base script for process hello_world
[2026-03-13 09:52:44] - INFO - hello_world|cb6bc9hopa :: Process hello_world started with PID: 33887

```

Each log entry generated by **jawm Process** follows this format:
```text
[DATE TIME] - LOG_LEVEL - PROCESS_NAME|PROCESS_HASH :: LOG_MESSAGE
```

# Example with dependency

A simple workflow with two dependent processes:

```python
import jawm

step1 = jawm.Process(
    name="initial",
    script="""#!/bin/bash
sleep 10
echo "Initial Process"
"""
)

step2 = jawm.Process(
    name="dependent",
    script="""#!/bin/bash
echo "Dependent Process"
""",
    depends_on=[step1]
)

step1.execute()
step2.execute() # alternatively `step2.execute([step1.hash])`
```

Here:

- `step2` will wait until `step1` finishes successfully.
- jawm ensures dependency order automatically.

# Process Execution

A process is executed using:

```python
p.execute()
```

During execution, jawm typically performs several steps:

1. **Parameter resolution**
2. **Variable expansion**
3. **Dependency validation**
4. **Logs directory creation**
5. **Script execution**
6. **Logging and monitoring**

---

# Logs and Artifacts


When a `Process` is executed, **jawm automatically creates a dedicated log directory** to store logs and execution artifacts.

The directory name follows this structure:

```text
<logs_directory>/<process.name>_<YYYYMMDD>_<HHMMSS>_<process.hash>
```

And inside that directory jawm writes process-specific files such as:
```text
<name>.script
<name>.command
<name>.output
<name>.error
<name>.id
<name>.exitcode
```

These files contain the generated script, launch command, standard output, standard error, process or job runtime identifier, and exit code.

There can be backend manager specific artifacts files as well, such as `<name>.slurm`, `<name>.k8s.json`; also `stats.json` if the stats recording is enabled.


In addition to per-process files, jawm also writes a central `error.log` file in the `logs_directory`. 
This file is intended as a **quick error summary**, so users can find failed processes without opening each individual process directory.

By default, it is stored at: `<logs_directory>/error.log`.

# Execution Backends

jawm supports multiple execution managers:

| Backend | Description |
|---|---|
| `local` | Execute directly on the local machine |
| `slurm` | Submit jobs to a Slurm cluster |
| `kubernetes` | Run jobs inside a Kubernetes cluster |

The manager can be configured per process.

---

# Execution Environments

Processes may also run inside containers.

Supported container systems include:

- Docker
- Apptainer / Singularity
